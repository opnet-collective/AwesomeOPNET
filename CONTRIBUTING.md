# Contribution Guidelines

Contributions are welcome! This is how you add something to Awesome OPNET.

---

## Adding or updating an entry

You will need a [GitHub account](https://github.com/join).

1. Go to the repository on GitHub
2. Click on `README.md` (or the relevant `docs/` file)
3. Click the edit (pencil) icon in the top right
4. Make your changes in the browser editor
5. Click **"Propose changes"** and describe what you're adding and why
6. Open the pull request

---

## What belongs here

To be included, a resource should:

- Be **publicly accessible** — open source, live app, or public documentation
- Be **specific to OPNET** — generic blockchain tools without OPNET integration don't qualify
- Be **live or have a known launch date** — clearly mark upcoming things as such
- Have a **clear, factual one-line description**

## What doesn't belong

- Private or unreleased projects with no public info
- Anything requiring special access or invite to use
- Pure marketing pages with no technical substance

---

## Entry format

Follow the existing table style:

```markdown
| **Project Name** | https://url.com | One sentence describing what it does |
```

- Use the project's real name as it appears publicly
- Link to the most useful page (app, docs, or repo — not just a landing page)
- If no URL is available yet, use `—`

---

## Fixing outdated information

If an address, URL, or description is wrong or stale, open a PR with the correction and include a source (announcement link, on-chain tx, or official docs).

---

## Questions?

Open an issue or ask in the [OPNET Discord](https://discord.com/invite/opnet).
